# Security Policy

## Reporting a Vulnerability

Please report suspected security issues privately by opening a private security advisory on GitHub or by contacting the maintainer directly through a non-public channel before public disclosure.

Do not include secrets, live tokens, or production database contents in your report.

## Response Expectations

The maintainer will aim to acknowledge new reports within 5 business days and will provide follow-up once the issue has been triaged.

## Supported Versions

Security fixes are currently targeted at the latest `master` branch and the latest deployed production baseline.

## Responsible Disclosure

Please allow a reasonable remediation window before public disclosure so a fix, workaround, or mitigation can be prepared and communicated safely.
